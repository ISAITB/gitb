@switch (tip) { @case (Constants.USAGE_TIP.TEST_BED_ADMIN_CREATE_CONFORMANCE_STATEMENT) {

You are about to create a conformance statement in the Test Bed administrator's organisation.

Conformance statements are normally created for organisations belonging to a community as this enables use of organisation-level and system-level configuration properties, and ensures community settings are applied. If your goal is to verify developed tests, you should do this through a community administrator account, or by using the "Manage tests" feature for a specific organisation in a community.

Nonetheless, conformance statements for the Test Bed administrator's organisation can still be useful for experimentation purposes, and as tests to ensure that your overall setup is working correctly.

} @case (Constants.USAGE_TIP.TEST_BED_ADMIN_CREATE_SYSTEM) {

You are about to create a system under the Test Bed administrator's organisation. You typically do this to test the system as part of one or more conformance statements.

Conformance statements are normally created for organisations belonging to a community as this enables use of organisation-level and system-level configuration properties, and ensures community settings are applied. If your goal is to verify developed tests, you should do this through a community administrator account, or by using the "Manage tests" feature for a specific organisation in a community.

Nonetheless, conformance statements for the Test Bed administrator's organisation can still be useful for experimentation purposes, and as tests to ensure that your overall setup is working correctly.

} @case (Constants.USAGE_TIP.TEST_BED_LANDING_PAGE) {

There is currently no default landing page set up.

The landing page is the first screen to greet users when they log in. It can be defined at the level of the overall Test Bed instance (in the system settings) or for a specific community (in the community settings). In terms of content, it can include text but also images and links to resources defined as system or community resources.

} @case (Constants.USAGE_TIP.TEST_BED_ADMIN_DOMAINS) {

The Test Bed uses domains to define and organise the specifications that users can test against, as well as the test cases to execute when specifications are selected for conformance statements.

Within domains you can also manage configuration properties that apply globally or per-specification. Note that configuration properties relevant to specific organisations and systems are managed within communities.

} @case (Constants.USAGE_TIP.TEST_BED_ADMIN_COMMUNITIES) {

The Test Bed uses communities for configuration related to user management. Communities include as organisations the parties that will be testing, within which you define users and the systems that will be tested.

Besides user management, communities are also used to define configuration properties, and to manage settings such as self-registration options, report templates, and custom landing pages.
